**SRS02:**

1. Глобальное использование using namespace std - во всех файлах присутствует директива, загрязняющая глобальное пространство имён. Требуется замена на явный префикс «std::».
2. Отсутствие префиксов у классов - классы и структуры не имеют обязательных префиксов «F», «U», «A» и т.д. (например, «Node», «Stack», «Queue», «LinkedList», «Spisok»). Необходимо добавить соответствующие префиксы.
3. Некорректный стиль именования - имена переменных, функций и методов не соответствуют PascalCase:
   - Имена в стиле camelCase или с подчеркиваниями
   - Использование транслита (например, «Dannye», «Predydushiy», «Perviy», «Uzel»)
   - Слишком короткие имена («n», «m», «x», «i», «V», «D», «L»)
4. Некорректное расположение скобок - открывающие фигурные скобки «{» расположены на той же строке, что не соответствует стилю Allman. Требуется перенос на новую строку.
5. Отсутствие пробелов вокруг операторов - в выражениях вида «i<n;i++» отсутствуют пробелы, что снижает читаемость.
6. Отсутствие деструкторов - во многих классах с динамической памятью нет деструкторов, что приводит к утечкам памяти.
7. Отсутствие обработки исключений - нет блоков try-catch для обработки ошибочных ситуаций.
8. Отсутствие проверок корректности ввода - ввод данных с клавиатуры не проверяется на корректность (например, ввод букв вместо чисел).
9. Отсутствие проверок открытия файлов - операции с файлами не проверяют успешность открытия.
10. Имена функций не отражают суть операций - некоторые функции названы неинформативно (например, «EvenHead», «MiddleElement», «IsNumber»).
11. Несколько переменных в одной строке - объявления вида «int n1, n2, n3;» снижают читаемость.
12. Отсутствие константности методов - методы, не изменяющие объект, не помечены как «const».
13. Отсутствие модульности - весь код находится в одном файле, нет разделения на заголовочные файлы и файлы реализации.
